title: 关于hexo的几个常见错误
date: 2021-10-23 19:37:54
tags:
​—
对于传博客，我们总是能与hexo的各种错误杠上。因此我今天于此总结一下hexo命令的几个错误已经解决方法。
1：当我们输入hexo d的时候出现：fatal: Authentication failed for ‘https://github.com/xxx/xxx.github.io.git /‘
1：git的密码输入错误（本质），当然在知道用户名密码的情况下都输错的低级错误那就不说了。
2：git的密码修改后而未及时修改为新的密码，导致出错。
3：git的账户被锁定
对此我们有以下几种解决方法：首先可以输入git config –list
（user.name=yuhua
user.email=xxx@xxx.com）此为需要对照的信息。
如果不对的话就需要重新配置。
git config –global user.name [username] 如：git config –global user.name yh
git config –global user.email [email] 如：git config –global user.email 111@163.com
如果是第二条的问题，我们则需要我们需要做如下操作（idea工具下操作）
1：在file->Settings->Passwords下在Do not save,forget passwords after restart前打勾。
2：在控制面板的凭据管理器中找到windows凭据，将git的密码修改成已经改过的新密码（操作完后保险起见可以重启一下机器）。
如果是第三条原因则需另请高明。。。。
2：atal: unable to access ‘https://github.com/a956551943/a956551943.github.io/': Encountered end of file
FATAL {
err: Error: Spawn failed
at ChildProcess. (/usr/local/src/hexo/hanyubolg/node_modules/hexo-util/lib/spawn.js:51:21)
at ChildProcess.emit (events.js:376:20)
at Process.ChildProcess._handle.onexit (internal/child_process.js:277:12) {
code: 128
}
} Something’s wrong. Maybe you can find the solution here: %s https://hexo.io/docs/troubleshooting.html
如果出线以上问题，则有以下几种方法：
1：可能是网络问题，这边建议换成手机热点不要使用校园网络。
2：删除.deploy_git文件夹;
输入git config –global core.autocrlf false
然后，依次执行：
hexo clean
hexo g
hexo d
3：有可能是你的git repo配置地址不正确,可以将http方式变更为ssh方式
##进入站点根目录
cd /usr/local/src/hexo/hanyubolg/
##删除git提交内容文件夹
vim _config.yml
##修改
deploy:
type: git
repo: https://github.com/yourname/yourname.github.io.git -> git@github.com:a956551943/weixiaohui.github.io.git
branch: master
##最后
hexo clean && hexo g && hexo d
4：强烈不推荐
##进入站点根目录
cd /usr/local/src/hexo/hanyubolg/
##进入depoly文件夹
cd .deploy_git/
##强制推送
git push -f
本次分享到此结束。
